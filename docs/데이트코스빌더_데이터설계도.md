# 데이트 코스 빌더 — 데이터 설계도

> ⚠️ **이 문서가 유일한 최신본이다** (마지막 갱신: 2026-07-16 — 스케치 30장 FINAL 확정 반영).
> 과거 버전은 Git 히스토리 참조. 변경사항은 별도 섹션에 쌓지 않고 **본문을 직접 고친다.**
> 제품 정의서(화면·기능)의 짝. 여기는 **"데이터를 어떤 모양으로 저장하나"**를 정리한다.

## 큰 그림 (한눈에)

```
AppData (앱에 저장되는 전부)
├ courses: Course[]              저장된 코스들
│    └ Course
│        ├ items: CourseItem[]     타임라인에 배치된 것 (순서 = 배열 순서)
│        ├ waiting: CourseItem[]   🧺 미배치 대기 (미배치째 저장 가능)
│        └ hotel: Place | null     호텔 (타임라인 밖 별도 슬롯)
├ drafts: Draft[]                임시본들 (만들다 만 것 — 여러 개 허용, 홈에만 표시)
├ favorites: Place[]             즐겨찾기 (찜 ★)
├ myRegions: string[]            내 지역 목록 (화면1.5 카드 = 이 목록)
├ coupleDate: string             사귄 날 (내정보 D+ 계산)
├ anniversaries: Anniversary[]   기념일들
├ todos: Todo[]                  할 일들
└ girlfriendInfos: { text: string }[]   여친 정보들
```

**핵심 개념 2가지:**
1. **타입 = 붕어빵 틀** (모양 정의). 타입 자체엔 실제 데이터가 없음.
2. **배열([]) = 틀로 찍은 걸 담는 상자.** 같은 틀(Place)로 찍어도 담는 상자(favorites / items)가 다르면 안 섞임.

## 타입 정의 (최신 확정본)

```typescript
type Place = {
  id: string;           // 구글 place_id 또는 직접 추가용 생성 id
  name: string;         // "국립과학박물관"
  category: string;     // "박물관" | "라멘" | "호텔" ...
  region: string;       // "우에노"
  rating: number | null;    // ⭐4.5 (직접 추가면 null)
  priceLevel: string | null; // "¥¥"
  photo: string | null;      // 대표 사진 URL
  website: string | null;    // 공식 사이트 (상세 [🌐] — 없으면 버튼 숨김, 예약 연결)
  url: string;               // 구글맵 페이지 (상세 [🗺️])
  memo: string;              // 내 메모 (상세에서 보기·수정)
  isManual: boolean;         // 직접 추가 여부
};

type CourseItem = {
  place: Place;
  memo: string;         // "공룡관 꼭 보기 · 13시쯤" — 시간 적고 싶으면 여기에
};                      // ★개별 시간(startTime/endTime) 없음 — 순서가 곧 코스 순서

type Course = {
  id: string;
  title: string;        // 저장 팝업에서 자유 입력 (빈칸 시작)
  date: string;         // "2026-07-11" — 예정/다녀온 자동 분류 기준 (오늘과 비교)
  startTime: string;    // 코스 전체 틀 "12:00" (화면1, 기본 12:00)
  endTime: string;      // "23:00" (기본 23:00)
  region: string;       // "우에노" — 코스 탭 카드에 📍표시
  items: CourseItem[];  // 배치된 것 (배열 순서 = 코스 순서)
  waiting: CourseItem[];// 🧺 대기 (조립·편집의 미배치 영역)
  hotel: Place | null;  // 타임라인 밖 별도 슬롯
};

type Draft = {
  course: Course;       // 만들다 만 상태 그대로
  lastScreen: string;   // 나갔던 화면 — 점선 카드 탭하면 여기서 재개
};

type Anniversary = {
  id: string;
  title: string;        // "여친 생일"
  date: string;         // "2026-07-20" — D-는 저장 안 하고 date로 계산
  repeat: boolean;      // true = 지나면 내년 D-로 / false = 지나면 자동 삭제
};

type Todo = {
  id: string;
  text: string;         // "생일 선물 예약하기"
  done: boolean;        // ☐/☑ 토글, 완료 = 핑크☑ + 취소선
};

type AppData = {
  courses: Course[];
  drafts: Draft[];
  favorites: Place[];
  myRegions: string[];
  coupleDate: string;
  anniversaries: Anniversary[];
  todos: Todo[];
  girlfriendInfos: { text: string }[];
};
```

## 데이터 흐름 (화면 ↔ 데이터)

- **홈 마이노트**: anniversaries·todos·girlfriendInfos를 map으로 표시. [+]팝업=추가, 탭→✏️=수정(같은 팝업에 값 채움), 🗑️=확인 후 삭제, ☐탭=done 토글
- **홈 예정 코스**: courses 중 date ≥ 오늘 + drafts(점선 카드, 맨 위)
- **화면1**: Course의 date·startTime·endTime 입력 (달력=월 이동 공용 부품)
- **화면1.5**: myRegions를 카드로 / 검색은 1회용(목록에 안 쌓임)
- **화면2·3**: 검색 결과(Place[])를 카드로. [+]담기 → waiting에 CourseItem으로 / ☆ → favorites 토글
- **화면4**: waiting(대기)과 items(타임라인)를 그림. 드래그 = waiting→items 이동·순서 변경. 카드 탭(편집)=memo 수정
- **화면5**: hotel에 Place 하나 (선택 방식 — 교체)
- **저장**: title 입력 → courses에 추가, drafts에서 제거
- **코스 탭**: courses를 date로 예정/다녀온 분류. 카드에 title·date·items수·📍region
- **즐겨찾기 탭**: favorites를 [장소][맛집][호텔] 필터(category로) / [+코스에 추가]→예정+임시본 목록→그 코스의 waiting으로
- **내정보**: coupleDate로 D+ 계산 / myRegions CRUD / 전체 초기화(확인 2번)
- **공유**: Course → 텍스트 + 구글맵 경유지 링크 생성

## 자주 헷갈리는 것 (기억용)

- **즐겨찾기 = 대기실**: Place만 모임. 코스에 들어갈 때 CourseItem(place+memo)으로 감싸짐.
- **D-데이는 저장 안 함**: dday 같은 필드 없음. 항상 date와 오늘을 비교해서 계산 (저장하면 어제의 D-16이 오늘도 16으로 남는 버그).
- **예정/다녀온도 저장 안 함**: date ≥ 오늘이면 예정. 필드가 아니라 계산.
- **count도 저장 안 함**: 장소 수 = items.length. (현재 가짜 데이터의 count: 4는 items 생기기 전 임시 — 알고 진 빚)
- **가짜 데이터 규칙**: 반드시 이 문서의 타입과 같은 모양으로. 화면에 표시 안 하는 필드(예: 홈 카드의 region)도 데이터에는 있어야 함.
