# 데이트 코스 빌더 — 데이터 설계도

> 제품 정의서(화면·기능)의 짝이 되는 문서. 여기는 **"데이터를 어떤 모양으로 저장하나"**를 정리한다.
> 화면 22개가 전부 이 구조 위에서 돌아간다.

---

## 큰 그림 (한눈에)

```
AppData (앱에 저장되는 전부)
├ courses: Course[]              내 코스들
│    └ Course
│        ├ items: CourseItem[]   배치된 칸들 (타임라인 블록)
│        │    └ CourseItem { place: Place, memo }  ※개별 시간 없음(순서로)
│        └ hotel: Place | null   호텔 (선택, 타임라인 밖)
├ favorites: Place[]             즐겨찾기 장소들 (찜)
├ anniversaries: Anniversary[]   기념일들
├ todos: Todo[]                  할 일들
└ girlfriendInfo: GirlfriendInfo[]  여친 정보들
```

**핵심 개념 2가지 (이것만 잡으면 됨):**
1. **타입 = 붕어빵 틀** (모양 정의). 타입 자체엔 실제 데이터가 없음.
2. **배열([]) = 틀로 찍은 걸 담는 상자.** 같은 틀(Place)로 찍어도 담는 상자(favorites / items)가 다르면 안 섞임.

---

## 타입 정의 (붕어빵 틀 6개)

### Place — 장소 하나
모든 장소의 기본 단위. 볼거리·맛집·카페·호텔 다 Place다.
```typescript
type Place = {
  name: string;        // 이름 - "국립과학박물관"
  rating: number;      // 평점 - 4.5
  hours: string;       // 영업시간 - "~17시"
  priceLevel: number;  // 가격 - 1~4단계 (구글 방식, 정확한 엔화 아님)
  category: string;    // 카테고리 - "볼거리"
  placeId: string;     // 구글 고유번호 - 이걸로 구글맵 링크·사진·상세 다시 불러옴
  memo: string;        // 유저 메모 - "공룡관 꼭 보기"
  photoUrl: string;    // 대표 사진 주소
  lat: number;         // 위도 (지도 핀·거리 계산용)
  lng: number;         // 경도
  isManual: boolean;   // 직접 추가한 장소인가? (구글에 없는 팝업·축제 등)
};
```
- **대부분 구글이 줌**: name·rating·hours·priceLevel·placeId·photoUrl·lat·lng 다 구글 Places가 한 번에 준다. 우리가 만드는 건 거의 없음.
- **유저가 만드는 것**: memo, 그리고 직접 추가 장소의 이름·위치.
- **placeId 주의**: 이름표(placeId)는 우리가 정하지만, 값("ChIJ...")은 구글이 붙인 고유번호. 도서관 청구기호 같은 것 — 이 번호로 그 장소를 다시 찾음.
- **isManual**: 직접 추가한 장소는 구글 데이터가 없어서(rating·placeId 등 비어있을 수 있음) 구분 표시가 필요.

### CourseItem — 코스 안의 한 칸 (타임라인 블록)
장소를 "언제 갈지"와 함께 감싼 것. 화면 4의 색깔 블록 하나.
```typescript
type CourseItem = {
  place: Place;        // 어떤 장소인지 (Place 통째로)
  memo: string;        // "공룡관 꼭 보기 · 13시쯤" — 시간도 적고 싶으면 여기에
};
```
- **★확정 변경 (화면4 대공사):** 개별 시간(startTime/endTime) **삭제.** 순서는 items 배열 순서가 곧 코스 순서. 시간을 적고 싶으면 memo에 ("13시쯤"). 이동 카드도 없음.
- **placed 구분:** 조립·편집 모드의 "🧺 대기(미배치)"와 "타임라인(배치됨)"을 나누기 위해 Course에 `waiting: CourseItem[]`을 별도로 둠 (미배치째 저장 가능).
- **비유**: Place=배우, CourseItem=배역. 같은 배우가 여러 드라마에 다른 배역으로.

### Course — 코스 하나
```typescript
type Course = {
  title: string;          // 이름 - "우에노 데이트" (저장 시 붙임)
  date: string;           // 날짜 - "2026-07-11"
  startTime: string;      // 데이트 전체 시작 - "12:00"
  endTime: string;        // 전체 끝 - "23:00"
  items: CourseItem[];    // 배치된 칸들
  hotel: Place | null;    // 호텔 (있으면 Place, 없으면 null)
};
```
- **두 종류의 시간 구분:**
  - `Course.startTime/endTime` = **데이트 전체 틀** (12~23시). 화면1에서 정함. 필터·타임라인 축 범위로 쓰임.
  - `CourseItem.startTime` = **그 안 각 장소 위치** (13시...). 회의로 치면 "전체 9~18시" vs "각 세션 시간".
- **hotel이 items 밖에 따로 (`Place | null`):** 호텔은 타임라인 밖 빈칸에 두기로 함(화면 설계). 그래서 데이터도 items(타임라인 안)와 hotel(밖) 분리. `| null` = 있을 수도 없을 수도(선택).
- **예정/다녀온은 저장 안 함:** date를 오늘과 비교해서 판단. 미래=예정, 과거=다녀온. 상태를 저장하면 매일 갱신해야 하므로 date만 둠.

### Anniversary — 기념일
```typescript
type Anniversary = {
  title: string;    // "여친 생일"
  date: string;     // "2026-07-20"
  repeat: boolean;  // 매년 반복? (생일=true, 100일=false, 유저가 선택)
};
```

### Todo — 할 일
```typescript
type Todo = {
  text: string;     // "생일 선물 예약하기"
  done: boolean;    // 체크했나 (완료 여부)
};
```

### GirlfriendInfo — 여친 정보
```typescript
type GirlfriendInfo = {
  text: string;     // "갑각류 알레르기"
};
```
- 화면 2·3에서 참고용으로 띄움 (자동 필터 아님, 유저가 봄).

## 앱 전체 데이터 (담는 상자)

```typescript
type AppData = {
  courses: Course[];                  // 내 코스들
  favorites: Place[];                 // 즐겨찾기 장소들 (찜)
  anniversaries: Anniversary[];       // 기념일들
  todos: Todo[];                      // 할 일들
  girlfriendInfo: GirlfriendInfo[];   // 여친 정보들
};
```

- **전부 독립적**: favorites가 courses 밖에 따로 있음. 코스를 다 지워도 favorites·anniversaries는 남는다. 각자 독립 목록.
- **즐겨찾기 = "코스에 넣기 전 대기실"**: 장소(Place)만 모여있다가, 코스에 들어갈 때 CourseItem(place+memo)으로 감싸짐.
- **이 AppData 통째로 localStorage에 저장.** (MVP. 저장 로직은 saveData/loadData로 추상화 → 나중에 백엔드 전환 시 함수 속만 교체)

## 데이터 흐름 (화면 ↔ 데이터)

- **화면1 (시간)** → Course의 date·startTime·endTime 채움
- **화면2·3 (장소·맛집)** → 구글 Places에서 Place 받아옴 → 담으면 코스의 items로, 찜하면 favorites로
- **직접 추가** → 이름·위치 입력한 Place 생성 (isManual: true)
- **화면4 (타임라인)** → items를 순서대로 블록으로 그림 (시간축은 배경 눈금). 드래그 = 순서 변경. 카드 탭(편집 모드) → memo 편집
- **화면5 (호텔)** → 고른 호텔을 Course.hotel에 넣음
- **저장** → 완성된 Course를 AppData.courses에 추가 → localStorage
- **즐겨찾기 → 코스**: favorites의 Place를 꺼내 CourseItem으로 감싸 **waiting(대기 영역)**에 추가 (즐겨찾기 탭 → 어느코스에? 팝업 / 화면2·3 → 담기 시트, 양방향)

## ★ V1 확정 반영 (스케치 30장 FINAL 시점 — 사양서와 동기화)

```typescript
type CourseItem = { place: Place; memo: string };   // 개별 시간 없음

type Course = {
  id: string;
  title: string;          // 저장 팝업에서 자유 입력 (빈칸 시작)
  date: string;           // "2026-07-11" — 예정/다녀온 자동 분류 기준
  startTime: string;      // 전체 틀 "12:00" (기본값, 화면1)
  endTime: string;        // "23:00"
  region: string;         // "우에노" — 코스 탭 카드에 표시
  items: CourseItem[];    // 배치된 것 (순서 = 배열 순서)
  waiting: CourseItem[];  // 🧺 미배치 대기 (미배치째 저장 가능)
  hotel: Place | null;    // 타임라인 밖 별도 슬롯
};

type Draft = {            // 임시본 (여러 개 허용)
  course: Course;         // 만들다 만 상태 그대로
  lastScreen: string;     // 나갔던 화면 — 재개 지점
};

type Anniversary = { id: string; title: string; date: string; repeat: boolean };
// repeat true = 지나면 내년 D-로 / false = 지나면 자동 삭제. dday는 저장 안 함(date로 계산)

type AppData = {
  courses: Course[];
  drafts: Draft[];        // ★추가
  favorites: Place[];
  myRegions: string[];    // ★추가 — 내 지역 관리 (화면1.5 카드 목록)
  coupleDate: string;     // ★추가 — 사귄 날 (내정보 D+ 계산)
  anniversaries: Anniversary[];
  todos: Todo[];
  girlfriendInfos: { text: string }[];
};

// Place 추가 필드: website?(공식 사이트 — 상세 [🌐], 없으면 버튼 숨김), url(구글맵), isManual?(직접 추가)
```
